# AI 奇摩股價預測與戰略分析系統｜專案說明

## 一、專案背景

在學習 Python、SQL、資料庫、機器學習與資料視覺化的過程中，我希望完成一個能整合所學、且具備實際操作流程的作品，因此建立這套台股分析系統。

專案不只呈現單一模型，而是從資料取得、資料庫建立、指標計算、模型訓練、股票篩選、視覺化儀表板到策略回測，完成一條相對完整的資料分析流程。

## 二、專案目標

1. 建立可維護的台股歷史資料庫
2. 將技術指標轉換為多空觀察訊號
3. 以機器學習預測隔日漲跌機率
4. 整合指標與模型結果建立股票篩選器
5. 使用 Streamlit 建立可互動的分析介面
6. 保存每次分析紀錄，方便日後回顧
7. 透過回測檢查策略的歷史表現

## 三、系統架構

### 子專案 0：資料取得與資料庫建立

負責建立 SQLite 資料庫，整理股票基本資料與歷史價格，作為後續所有分析的資料來源。

### 子專案 1：多空指標計算

根據價格、成交量、均線、RSI、KD、MACD 等欄位，產生多空分數與訊號分類。

### 子專案 2：股價預測

以技術指標作為模型特徵，建立隔日漲跌分類模型，輸出上漲機率、模型報告與特徵重要性。

### 子專案 3：股票篩選器

整合多空分數與預測機率，將股票分為強勢看多、保守觀察、風險偏高與一般觀察。

### 子專案 4：戰略儀表板

使用 Streamlit 與 Plotly 建立互動式介面，提供股票搜尋、時間區間、技術圖表、策略摘要與分析紀錄。

### 子專案 5：策略回測

根據歷史訊號模擬策略結果，輸出交易紀錄、資產曲線與回測摘要。

## 六、專案挑戰與解法

### 1. 資料欄位格式不一致

部分價格欄位包含逗號、空值與特殊符號，因此先進行清理與數值轉換，確保技術指標可以正確計算。

### 2. 股票代碼前導零問題

例如 0050 容易在讀取資料時變成 50，因此統一將股票代碼轉為字串並補足四碼。

### 3. 多個子專案輸出需要整合

不同模組輸出 CSV、模型與資料庫檔案，因此統一路徑與欄位名稱，並加入安全讀取與檔案狀態檢查。

### 4. 儀表板功能與介面整合

在保留原有分析功能的前提下，加入股票搜尋、自訂 Range、一鍵分析、分析紀錄與深色莫蘭迪介面。

### 5. 分析結果需可回顧

新增分析歷史紀錄功能，讓使用者可以重新載入先前的分析結果。

## 七、模型結果與限制

目前第一版模型準確率約為 52.53%，表示模型僅略高於隨機猜測。股價會受到政策、消息、籌碼與市場情緒影響，因此本系統不應作為單一投資依據。
